# Routing Artifact Prevalence Table

## Overview

This document reports the marginal prevalence of each artifact class in the
routing namespace, measured over benign GitHub interaction logs.

Prevalence is defined as **raw access-event counts per artifact class** within a
specified observation window. These statistics characterize how frequently
different artifact classes are *accessed or acted upon* in normal platform
activity and do not encode behavioral structure, user intent, or routing paths.

This document is **structural and population-level**. It does not assess
whether specific users behave suspiciously.

---

## Measurement Definition

### Time Window Selection

Artifact prevalence statistics are computed over a fixed, closed historical
window rather than a rolling or real-time interval.

**Time window:** June 1, 2025 → July 1, 2025

This window was chosen to satisfy the following criteria:

- It reflects contemporary GitHub usage patterns relevant to modern
  collaborative development workflows.
- The interval is fully completed, avoiding partial ingestion, delayed
  event reporting, or incomplete hourly archives.
- The one-month duration is sufficient to smooth short-term fluctuations
  in activity while remaining focused on structural artifact prevalence
  rather than long-term platform evolution.
- Using a fixed historical window ensures reproducibility and prevents
  confounding effects from ongoing platform changes or measurement drift.

The purpose of this window is not to model temporal trends or seasonality,
but to establish a representative baseline for the **marginal frequency of
artifact access events** in benign platform logs.

---

### Unit of Measurement

Each count corresponds to a **single logged GitHub event** associated with
an action on an artifact instance in benign platform logs.

An *access event* is defined as:

- A recorded interaction indicating that a user created, modified, or
  commented on an artifact instance.
- Events represent **observable actions**, not the full lifecycle or state
  of an artifact.

---

### Counting Semantics

- Each event is counted independently.
- Multiple events referencing the same artifact instance are counted
  separately.
- Events generated by the same user or different users are treated
  identically.
- No aggregation or normalization is performed at the user level.

These counts are **event-centric proxies** for artifact prevalence and do not
enumerate all possible interactions with an artifact class.

---

### Included Artifacts

Only artifact classes defined in the routing namespace are included:

{ Repository, Issue, PullRequest, Commit,
IssueComment, PullRequestConversationComment,
PullRequestReviewComment, CommitComment }

Administrative or metadata-only events (e.g., stars, forks, membership changes)
are excluded.

---

## Data Sources

- **Primary source**:
  - GH Archive (public GitHub event stream)

- **Dataset version**:
  - GH Archive daily tables, snapshot covering 2025-06-01 through 2025-07-01

- **Population scope**:
  - Public repositories
  - All users
  - All event producers (no filtering by role or permission level)

---

## Event-to-Artifact Mapping

GitHub events represent **specific actions on artifacts**, not artifacts
themselves. Each event type is mapped to the artifact class it operates on.

| Event Type | Artifact Class |
|----------|----------------|
| PushEvent | Commit |
| PullRequestEvent | PullRequest |
| IssuesEvent | Issue |
| IssueCommentEvent | IssueComment |
| PullRequestReviewEvent | PullRequestConversationComment |
| PullRequestReviewCommentEvent | PullRequestReviewComment |
| CommitCommentEvent | CommitComment |

Events outside this mapping are excluded from prevalence counts.

## Column Definitions

### Artifact Class
Name of the artifact class as defined in the routing namespace.

### Raw Count
Total number of logged GitHub events mapped to this artifact class within
the observation window.

### Relative Frequency
Normalized prevalence computed as:

raw_count / Σ(raw_count across all artifact classes)

Relative frequencies sum to 1.0 across all artifact classes.

---

## Intended Use

The prevalence table is used to:

- Characterize baseline routing surface availability
- Inform discussion of artifact selection realism
- Support sensitivity and ablation analyses
- Ground routing assumptions in empirical platform behavior

These values are **not** used to:
- train behavioral models
- infer user intent
- detect covert communication

---

## Limitations

- Event logs capture only a subset of possible artifact interactions.
- Some artifact actions do not emit events.
- Counts depend on GH Archive coverage and logging fidelity.
- Results may not generalize across platforms, repository types, or time periods.
